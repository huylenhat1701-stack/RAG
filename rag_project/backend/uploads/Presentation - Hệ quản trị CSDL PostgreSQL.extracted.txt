mẽ.

Các lệnh DDL cơ bản
CREATE TABLE
Lệnh CREATE TABLE cho phép tạo bảng mới
trong cơ sở dữ liệu với các cột và kiểu dữ liệu
xác định.
ALTER TABLE
Lệnh ALTER TABLE được sử dụng để thay đổi
cấu trúc bảng đã có, như thêm, sửa đổi hoặc
xóa cột.
DROP TABLE
Lệnh DROP TABLE xóa bỏ bảng và tất cả dữ
liệu liên quan, cần thực hiện cẩn thận để
tránh mất dữ liệu quý giá.

Các lệnh DML cơ bản
INSERT
Lệnh INSERT INTO dùng để thêm dữ liệu mới
vào bảng, có thể thêm nhiều bản ghi cùng lúc
với cú pháp đơn giản.
UPDATE
Lệnh UPDATE cho phép sửa đổi dữ liệu đã có
trong bảng, với khả năng cập nhật nhiều bản
ghi cùng một lúc dễ dàng.
DELETE
Lệnh DELETE được sử dụng để xóa các bản
ghi nhất định khỏi bảng, có thể chỉ định xóa
theo điều kiện cụ thể.

Các lệnh nâng cao
JOIN
JOIN cho phép kết hợp dữ liệu từ hai bảng
hoặc hơn, giúp truy vấn trở nên linh hoạt và
mạnh mẽ hơn.
INDEX
Việc tạo index giúp tối ưu hóa hiệu suất truy
vấn bằng cách giảm thời gian tìm kiếm dữ
liệu trong bảng lớn.
CODE MẪU
Dưới đây là đoạn code mẫu minh họa cách
sử dụng JOIN và tạo index trong PostgreSQL,
giúp cải thiện hiệu suất truy vấn.

Quản lý Người dùng và Role
CREATE USER
Lệnh CREATE USER cho phép bạn tạo người
dùng mới trong PostgreSQL với các quyền
hạn cụ thể cho từng user.
CREATE ROLE
Lệnh CREATE ROLE cho phép bạn tạo các role
trong PostgreSQL, giúp quản lý quyền truy
cập một cách hiệu quả hơn.
PHÂN LOẠI USER/ROLE
PostgreSQL hỗ trợ phân loại user và role
thành superuser và regular user, đảm bảo an
toàn và bảo mật cho cơ sở dữ liệu.

Phân quyền trong PostgreSQL
GRANT
Lệnh GRANT cho phép cấp quyền truy cập dữ
liệu cho người dùng hoặc role cụ thể trong
hệ quản trị cơ sở dữ liệu.
REVOKE
Lệnh REVOKE được sử dụng để thu hồi quyền
đã cấp trước đó cho người dùng hoặc role,
giúp bảo mật cơ sở dữ liệu.
TÁC ĐỘNG
Việc phân quyền chính xác giúp bảo vệ dữ
liệu nhạy cảm và đảm bảo rằng chỉ những
người dùng có quyền mới có thể truy cập.
